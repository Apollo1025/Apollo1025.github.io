前言

在刚开始想搭建博客的时候，我像很多人一样选择了老字号WordPress，接下来的一大套，买服务器，买域名，实名，备案……时间周期长，弯路走得多，搞得脑阔子疼，最后发现了hexo，搭建的时候很简单，不买域名的话是真的零成本，缺点是写博客比较麻烦，而且访问速度比较慢，不过依然是瑕不掩瑜。

在用hexo的时候，依然走了很多弯路，花了一天的时间，记录一下搭建过程，避免同一个坑跳两次顺便写个保姆级教程（雾）？

正文

注意，如果此文中提供的指令输入后出错，那么建议你切换英语键盘来重新输入一遍

准备工作

1、下载node.js

网址：https://nodejs.org/en/

进入官网后点击箭头所指下载即可

下载完后打开安装包除了设置安装路径（如果你不想按默认路径的话）其他的不停的点下一步就行了

安装完成后关闭安装界面

2、下载安装git

网址：https://git-scm.com/downloads

进入后选择相应系统的安装，我这是Windows，所以根据箭头上的选择就可以了

之后选择下载，运行，还是一直点下一步就行了

准备工作到此结束

3、申请一个github账号

网址：https://github.com/

起名最好都是小写英文，别用中文，填入相关资料申请就可以了，免费的

4、下载typora（如果你有更好的markdown编辑器也可以）

地址：https://www.typora.io/

下载Windows即可，安装也是除了想自定义安装路径，点下一步就行了

正式开始

部署到本地

首先在你的电脑中，右键空白处，新建一个文件夹，我这里在D盘新建的文件夹1起名叫做blog

之后进入你新建的这个文件夹，右键空白处，选择git bash here

之后输入以下指令，检查node.js以及npm是否安装成功

之后输入以下指令

npm install -g cnpm --registry=https://registry.npm.taobao.org

安装cnpm用于提高速度，安装完成后输入cnpm，出现下图这种即为成功

之后安装我们的主角hexo，输入以下指令

cnpm install -g hexo-cli

等待再次出现新的一行如下图所示，在安装完成后才能再输入指令，未出现如下图中新的一行代表还在安装中，稍安勿躁

之后输入

hexo -v

出现下图这种即为成功

之后我们运行命令

hexo init

等待完成，安装完成后，你的文件夹是这样

这样基本上就大功告成了

在你每次更新博客的时候，都要用上以下命令，注意先后顺序

hexo c用来清理缓存文件

hexo g生成文件

之后是两个命令，有不同的功能

hexo s运行到本地服务器，一般用来看看效果怎么样

hexo d现在还用不到，等会讲

在我们输入hexo s后，他会如下图所示

拷贝，粘贴进浏览器后，就可以看到你的博客啦

之后我们在git bash中按ctrl+c退出，不然无法输入别的指令

部署到远端

现在这个网页还只能自己看到，那么怎么能让别人也看到你的网站呢？我们就需要github的网页托管功能了（白嫖大法好）

登录我们的github账号，点击那个小加号，选择 New repository（新建仓库）

之后注册一个仓库，给他命名，这里注意！这里注意！这里注意

仓库的命名一定要是 你的名称.github.io

比如说我的名称是箭头所指的Apollo1025，所以我的仓库名为apollo1025.github.io

Description是描述，随便写一下就行了

然后点击Creat repository创建仓库

之后返回我们的gitbash，输入以下指令

cnpm install --save hexo-deployer-git

进行安装，出现warning不用管，忽略就行

之后我们配置一下_config.yml

打开你刚刚创建的仓库，复制https地址

拉倒底部，在type后输入git，repo后粘贴你的github仓库地址，branch后输入master，注意：type： repo：等等冒号后面一定要加一个空格再写，不然会出错

保存退出就行了

之后输入指令 hexo d

他可能会让你输账号，密码，输入你的github账号密码就行了

之后再浏览器输入你的http://加仓库名就可以访问了

比如说我的仓库名是apollo1025.github.io

在浏览器输入apollo1025.github.io就可以访问了，如果不能访问，刷新一下，如果还不行就稍等一下，吃个饭打局农药什么的再试试

可以访问了的话你可以把他发给你的朋友了

更新文章

当你想更新一篇文章的时候,输入指令hexo n "你的文章名称"

创建完成后在你根目录的source/_posts文件夹中打开你的文章，写就行了

写完后还是在gitbash中输入指令

hexo c

hexo g

之后要是想在本地查看效果

输入

hexo s

没问题后想提交给远端的话，先在gitbash中按住ctrl+c退出本地在输入hexo d提交给远端就行了

缺失模块。
1、请确保node版本大于6.2
2、在博客根目录（注意不是archer根目录）执行以下命令：
npm i hexo-generator-json-content --save
3、在根目录_config.yml里添加配置： 
jsonContent:
  meta: false
  pages: false
  posts:
    title: true
    date: true
    path: true
    text: false
    raw: false
    content: false
    slug: false
    updated: false
    comments: false
    link: false
    permalink: false
    excerpt: false
    categories: true
    tags: true
